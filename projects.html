Raspberry Pi 5 Home Server

Ignoring the currently (still) high prices, the Raspberry Pi 5 is a super powerful PC, even though it has the size of a credit card. Whether you use it as a desktop PC, a gaming console, or like me, a home server, you'll find so many amazing use cases. Here I will showcase some things I have done and show you one or the other setup.

This project covers the setup of a Raspberry Pi 5 as a small home server, including installation of services like web servers, monitoring tools and Docker.

What you'll need:

  • Raspberry Pi 5
  • MicroSD card (at least 32GB or Nvme SSD)
  • Power supply
  • Internet connection (Ethernet or WiFi)
  • (Optional) Keyboard and mouse
  • (Optional) HDMI cable for monitor connection
  • (Optional) Raspberry Pi 5 Case

My configuration includes the Argon ONE V3 NVME Case and a 256GB NVMe SSD, which was repurposed from my laptop after an upgrade. This case offers a built-in an for thermal management and an NVMe SSD adapter. While the adapter adds to the cost, an SSD is in all ways superior to a MicroSD, especially when you consider setting up something like a remote file system.
This thing with my power supply is pretty much the hardware setup. The rest is configuration on the Pi itself. Don't worry, pretty much all of the services I installed have a GUI to them, making it super easy to configure things, without living in the terminal emulator.

Initial Setup
1. Install your operating system. I recommend booting from an SSD for better performance. You can set this up using a bootable USB or MicroSD card
2. My OS of choice is Linux Server, which operates entirely via the console (no desktop environment), but is not needed, just more lightweight
3. Configure SSH (Secure Shell) to enable remote access to your Raspberry Pi (you can look that up)

Networking Basics
Some things you need to know:
  • Your router’s IP address (commonly 192.168.1.1) serves as a gateway to your network settings
  • Tools like Fing (a mobile app) can scan your network and list connected devices. Alternatively, access your router’s GUI through a browser for similar information
  • Some routers, like mine, even offer features such as WireGuard VPN, eliminating the need to set up VPN services on the Raspberry Pi

What you basically need to know is the addressing scheme of your private network. Because all the other devices will have the same address space, just a different number at the end to uniquely identify you devices.

What I installed
  • Monitoring: RPI Monitor, Grafana + Prometheus
  • Docker Management: Portainer
  • File Sharing/Cloud: NextCloud
  • Distributed Computing: folding@home
  • News Aggregator: TTRSS (Tiny Tiny RSS)
  • Dashboard: Homarr

Most of the applications in this list are deployed as Docker containers, which is a super easy way to install applications but also manage them.

Kubernetes with Raspberry Pi?!

This project builds on the RPI5 home server project by adding some spicy hardware. A Raspberry Pi 4, and four, yes four Raspberry Pi Zero 2W. Everything fitted neatly into a cluster case.

This project demonstrates how to set up a Raspberry Pi cluster using Kubernetes. The goal is to turn the cluster into a high-performance computing system that can run containerized applications. It explores the challenges of managing multiple Raspberry Pi units and how Kubernetes can help.

Switching to Arch Linux + KDE Plasma/Hyprland

After having used Windows, MacOS and Linux Mint, its time for Arch Linux. Quite an impulsive decision, but so far it is probably my best decision so far. I'll guide you through some things here from a perspective of a Linux beginner.

To start things off, I did the manual install. So no ArchInstaller. I used dual boot Windows 11 and Linux Mint before, but ended up completely wiping the SSD because I messed up something. So I started completely from scratch essentially. Your best friend with Arch Linux is going to be the Arch Wiki. Super easy to follow and also has pretty much anything you'll ever need in your journey.
When I installed it, I still decided to dual boot, but with Mint. Basically because I was worried that Arch would be too hard for a beginner like me, or that I would mess up something and lose all my data. Always back up your data . It ended up not being too much of a problem because of all the resources out there.

How did I start now?
Just follow the installation wiki. But to sum it up: Create boot USB with Arch ISO and boot via USB, create partition, root and efi for UEFI boot and format them accordingly. Then set up your network and download some stuff and finally chroot into your filesystem. At this point you just need a GUI and you're pretty much done.
In terms of GUI I started using SDDM with KDE Plasma to start things off. Then I discovered Tiling (with Hyprland). And its my absolute favourite right now. Theres also a description on the wiki, but basically we have something like Windows, where your application windows are floating. You can pull them around and they can be on top of each other and all that typical stuff. What Tiling is, is that you don't have this anymore. Each window is a tile on your display. You can resize, but you cannot overlap those, like you can with Windows. A lot of stuff ends up being keyboard shortcuts and you'll definitiely start feeling like a hackerman.
