Raspberry Pi 5 Home Server

Ignoring the currently (still) high prices, the Raspberry Pi 5 is a super powerful PC, even though it has the size of a credit card. Whether you use it as a desktop PC, a gaming console, or like me, a home server, you'll find so many amazing use cases. Here I will showcase some things I have done and show you one or the other setup.

This project covers the setup of a Raspberry Pi 5 as a small home server, including installation of services like web servers, monitoring tools and Docker.

What you'll need:

  • Raspberry Pi 5
  • MicroSD card (at least 32GB or Nvme SSD)
  • Power supply
  • Internet connection (Ethernet or WiFi)
  • (Optional) Keyboard and mouse
  • (Optional) HDMI cable for monitor connection
  • (Optional) Raspberry Pi 5 Case

RPI5 with Argon Case


My configuration includes the Argon ONE V3 NVME Case and a 256GB NVMe SSD, which was repurposed from my laptop after an upgrade. This case offers a built-in an for thermal management and an NVMe SSD adapter. While the adapter adds to the cost, an SSD is in all ways superior to a MicroSD, especially when you consider setting up something like a remote file system.

This thing with my power supply is pretty much the hardware setup. The rest is configuration on the Pi itself. Don't worry, pretty much all of the services I installed have a GUI to them, making it super easy to configure things, without living in the terminal emulator.

Initial Setup
1. Install your operating system. I recommend booting from an SSD for better performance. You can set this up using a bootable USB or MicroSD card
2. My OS of choice is Linux Server, which operates entirely via the console (no desktop environment), but is not needed, just more lightweight
3. Configure SSH (Secure Shell) to enable remote access to your Raspberry Pi (you can look that up)

Networking Basics
Some things you need to know:
  • Your router’s IP address (commonly 192.168.1.1) serves as a gateway to your network settings
  • Tools like Fing (a mobile app) can scan your network and list connected devices. Alternatively, access your router’s GUI through a browser for similar information
  • Some routers, like mine, even offer features such as WireGuard VPN, eliminating the need to set up VPN services on the Raspberry Pi

What you basically need to know is the addressing scheme of your private network. Because all the other devices will have the same address space, just a different number at the end to uniquely identify your devices.

What I installed (in no particular order)
  • Monitoring: RPI Monitor, Grafana + Prometheus, Uptime Kuma
  • Docker Management: Portainer
  • File Sharing/Cloud: NextCloud
  • Distributed Computing: folding@home
  • News Aggregator: TTRSS (Tiny Tiny RSS)
  • Dashboard: Homarr
  • Document Management: Paperless
  • Media: Jellyfin

Most of the applications in this list are deployed as Docker containers, which is a super easy way to install applications but also to manage them. Management works either with Portainer, a GUI for your Docker containers, or simply by using the terminal. I end up using both, depending on what is just more comfortable for me at the moment.

Installation Process
In order to start installing things, it is always good practice to update your pi with: 

sudo apt update && sudo apt upgrade -y

You can also just use the two commands one after another. The -y flag automatically says yes for you, since apt upgrade should always ask you whether you want to install the upgrades or not. This should take some time but afterwards you're ready to go.
When I started out, I firstly installed RPI Monitor. A super simple tool, but its quite nice to see your RAM and CPU usage. After that I installed Docker with: 

sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

This command was taken from the Docker Website: https://docs.docker.com/engine/install/ubuntu/. Pretty much all things I host, I'll host via a Docker container deployment. Docker is amazing because you can containerise anything. Thats why I am able to host so many useful things, by simply setting up the respective container and I'm good to go. This also makes it quite easy to deploy containers, because it doesn't really matter on which machine you're on. Windows or Linux, you can pull the Docker image for your desired application and you can deploy it.
At this point here I also have to mention the Docker Hub: https://hub.docker.com/. This is the marketplace for your container images. If you need something, you can just go there and make a quick search. Usually, images will link the GitHub for the source code with installation and configuration steps, sometimes you just need to do a quick google search.

Kubernetes (K8's) with Raspberry Pi?!

This project builds on the RPI5 home server project by adding some spicy hardware. A Raspberry Pi 4, four, yes four Raspberry Pi Zero 2W. Everything fitted neatly into a cluster case.

First of all, what is Kubernetes and what are cluster and most importantly, why all of that?
In simple words, starting with clusters, clustering means that you basically have multiple PC's which work as one, while you have one master PC controlling those workers nodes. You are not going to combine multiple processors to achieve a giga processor, but rather you are able to split work evenly across your computation devices. Basically Docker, but with PC's.

What do you need for that you may ask? That's what Kubernetes is for, also called K8's. Kubernetes is a container orchestration tool, with which you can manage what I just described. Specifically for the RPI setup I am using K3's, which is K8's, but more lightweight, so our Pis can handle it. The installation of K3's itself is super simple. This time the most challenging thing was to set up my 5 Pis, the RPI4, and the four Zero 2 W's.

Switching to Arch Linux + KDE Plasma/Hyprland

After having used Windows, MacOS, and Linux Mint, it's time for Arch Linux. Although quite an impulsive decision, it has probably been my best choice so far. I'll guide you through the process from the perspective of a Linux beginner.

To start things off, I did a manual install—no ArchInstaller. I had previously used dual boot with Windows 11 and Linux Mint but ended up completely wiping my SSD after making a mistake. I started fresh from scratch. Your best friend when working with Arch Linux will be the Arch Wiki. It's super easy to follow and contains pretty much everything you'll need on your journey.

When I installed Arch, I decided to dual-boot with Linux Mint. This was mostly because I was worried Arch would be too complicated for a beginner like me, or that I might mess something up and lose all my data. Always back up your data. Fortunately, it wasn't as difficult as I thought, thanks to all the resources available online.

How did I start?
I followed the installation wiki, but to summarize: First, I created a bootable USB with the Arch ISO and booted from it. Then, I created partitions for root and EFI (for UEFI boot), formatted them accordingly, set up the network, and finally chrooted into the filesystem. After this, all I needed was a GUI.
I started with SDDM and KDE Plasma, but later discovered Tiling with Hyprland, which is now my absolute favorite. The difference between a tiling window manager and a traditional floating window system (like in Windows) is that, in a tiling manager, application windows are arranged as tiles. You can resize the tiles, but they cannot overlap. A lot of navigation happens via keyboard shortcuts, which makes you feel like a true hackerman.

My current Hyprdot Setup



This Hyprdots setup was made by Prasanthrangan, see for yourself on GitHub . Its an amazing configuration and helps me as a beginner to get into "Ricing".

What is "Ricing" you may ask? One word, customization. With Arch Linux the desktop environment can look like whatever you want it to look like.
